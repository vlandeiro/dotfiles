1. **Sixel graphics** ÔÇö Some terminals (including newer ghostty versions) support Sixel image protocol. tmux can pass through Sixel sequences if configured:
   ```
   set -g allow-passthrough on
   ```
   This lets you pipe images through tools like `img2sixel` or `chafa` to display them in tmux.

2. **Image preview tools** ÔÇö Use tools that render images as text:
   - `chafa` ÔÇö Converts images to ANSI/Unicode art
   - `viu` ÔÇö Shows images in the terminal using blocks/colors
   - `imgcat` ÔÇö iTerm2-style image rendering (if your terminal supports it)

3. **External viewers** ÔÇö Copy file paths into tmux and open them with an external image viewer:
   ```bash
   open image.png  # macOS
   ```

## Agent (2026-02-27 10:00:34)

The tmux config is typically at one of these locations:

1. **`~/.tmux.conf`** ÔÇö The default location tmux checks first
2. **`~/.config/tmux/tmux.conf`** ÔÇö XDG Base Directory spec location (if you prefer organizing in `.config/`)

tmux checks them in this order:
- `~/.tmux.conf` (if it exists)
- `~/.config/tmux/tmux.conf` (if `~/.tmux.conf` doesn't exist)

**On macOS specifically**, tmux will look in:
- `~/.tmux.conf`
- `~/.config/tmux/tmux.conf`

Since you're using dotfiles with a `.config` structure, you probably want to use `~/.config/tmux/tmux.conf` for consistency. You can verify which one you have:

```bash
ls ~/.tmux.conf ~/.config/tmux/tmux.conf
```

If you don't have one yet and want to follow your existing dotfiles structure, create it at `~/.config/tmux/tmux.conf`.
